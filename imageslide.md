## 1. 개요 (Overview)
이 프로젝트는 복잡한 JavaScript 애니메이션 라이브러리 없이 **순수 CSS(Pure CSS)**만으로 3D 공간감이 느껴지는 이미지 캐러셀(Carousel)을 구현한 사례입니다. 최소한의 코드 작성으로 양옆으로 갈수록 흐려지는 시각적 효과와 부드러운 3D 회전을 만들어낸 것이 특징입니다.

## 2. 주요 프론트엔드 기술 및 기능 분석

### 2.1. CSS (핵심 시각화 및 3D 렌더링)
이 프로젝트의 시각적 인터랙션은 대부분 CSS를 통해 통제됩니다.

* **3D 원근감 부여 (`perspective`)**
    * `.scene` 클래스에 `perspective: 35em;`을 적용하여 2D 평면을 3D 공간으로 변환했습니다. 이 값이 작을수록 원근 효과(왜곡)가 극대화되어 사용자와 가까운 이미지는 더 크게, 먼 이미지는 더 작게 보이게 만듭니다.
* **페이드아웃 마스크 효과 (`mask`)**
    * `.scene` 컨테이너에 `mask: linear-gradient(90deg, #0000, red 20% 80%, #0000);`를 사용했습니다. 
    * 이를 통해 캐러셀의 양쪽 끝(0~20% 및 80~100% 구간)으로 갈수록 이미지가 서서히 투명해지는 자연스러운 페이드(Lateral fade) 효과를 구현했습니다. 시선이 중앙에 집중되도록 유도하는 훌륭한 UI/UX 기법입니다.
* **그리드 레이아웃 (`display: grid`)**
    * `.scene`과 `.a3d` 요소에 `display: grid`를 적용하고, 내부 요소에 `place-self: center;`를 사용하여 까다로운 중앙 정렬을 단 한 줄의 코드로 간결하게 해결했습니다.

### 2.2. JavaScript (데이터 관리 및 보조)
JavaScript는 애니메이션이나 DOM을 직접 조작하는 대신, 렌더링할 데이터의 상태를 정의하는 데 사용되었습니다.

* **이미지 데이터 배열화**
    * `DATA` 상수에 해파리(Jellyfish) 사진들을 불러오기 위한 고유 이미지 코드들을 배열로 저장했습니다.
* **동적 렌더링 준비**
    * `const N = DATA.length;`를 통해 렌더링해야 할 아이템의 총개수를 계산합니다. 이 데이터는 HTML 템플릿 엔진(예: Pug)이나 CSS 변수(Custom Properties)와 결합하여 3D 원을 구성하는 각도(360도 / N)를 계산하는 데 활용됩니다.

## 3. 요약 및 인사이트
* **성능 최적화:** 무거운 JS 연산 대신 브라우저의 하드웨어 가속을 쉽게 받을 수 있는 CSS의 `transform`과 `perspective`를 활용하여 애니메이션의 프레임 드랍을 최소화했습니다.
* **모던 CSS 활용:** `mask` 속성이나 `place-self` 같은 모던 CSS 속성을 적극적으로 사용하여 불필요한 마크업(HTML 래퍼 요소 등)을 대폭 줄인 "미니멀 코드"의 정석을 보여줍니다.
